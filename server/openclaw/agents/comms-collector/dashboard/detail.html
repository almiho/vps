What Happens When Invoked

  1. Initialises MCP session with the Google Workspace wrapper (port 8091)
  2. Calls search_gmail_messages with query is:unread (up to 50 results)
  3. For each message found, fetches full content via get_gmail_message_content
  4. Normalises each message to the bus schema: source channel, sender, subject, body, reply context
  5. Writes new messages to data/bus.db with status = 'pending'
  6. Skips messages already present (deduplication by thread ID)
  7. Updates agents/comms-collector/dashboard/status.json with run results

The Comms Collector does not process, prioritise, route, or reply to messages. That is the Chief of Staff's responsibility (Milestone 5).
