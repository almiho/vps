Mission
Pure router — reads every pending message from the bus and decides which domain agent owns it. Nothing else.
Role in the system
Keyword-matches on subject, sender and first 200 chars of body. Clear match → domain_tag set, status='tagged'. Ambiguous → status='needs_review' + CoS review message written to bus. Logs every decision to decisions.jsonl.
Why it matters
Without a router, every domain agent would have to scan all pending messages. Inbox Manager gives each specialist a clean, pre-filtered queue and prevents duplication across agents.
Connects to
Reads pending messages from the bus (bus.db). Writes domain_tag and status back. Writes needs_review notifications to CoS (domain_tag='cos') on the bus.
Does NOT overlap with
Inbox Manager never analyses content, sets priorities, or suggests actions. It only routes. All downstream intelligence lives in the domain agents and CoS.
